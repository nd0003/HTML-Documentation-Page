Introduction To HTML

HTML is the language that every website on the internet is written in (including this one). HTML stands for HyperT ext Markup language and was created by a group named WHATWG in 1993. HTML on its own is not very useful however, as Cascading Style Sheets (CSS) needs to be applied in order for the website to look nice. While CSS is important, we will mainly be focusing on HTML here.
Similar to all Standard Generalised Markup Languages (SGMLs), HTML uses opening tags (<tag>) and closing tags (</tag>). A commonly used tag in HTML is the <p> tag. This tag is used to display text on a website. Below is an example of how the <p> tag is used in HTML.
<p>This is an example of an HTML paragraph.</p>
When a website's code has been written and the website is ready to go on the internet, the HTML and CSS are published to a web-host. This means that the code gets sent to the URL and when you visit the website you will be able to see the content of it, as the browser has processed the code for you.

Text Editors

HTML is written in a software called a text editor. Most computers have built-in text editors, but there are more advanced text editors used by web developers to make things easier for them. When practicing HTML for the first time, it is recommended to use a basic text editor like Notepad on Windows, or TextEdit for MacOS (Notepad come with Windows for free and TextEdit comes with MacOS for free). It is recommended to use these when starting out because you have to manually type out everything, and you will learn much better from your mistakes. Once you have learnt at least the basics of HTML, (Important tags, how the language works, etc) you can switch to a more advanced text editor if you wish to, but it is not required. Below is a list of text editors which HTML can be written in.

  • Notepad
  • TextEdit
  • Visual Studio Code
  • Sublime Text
  • Notepad++
  • Atom
  • CoffeeCup
  • DreamWeaver
You're probably also wondering which one my favourite is, and which one I recommend. As I said before, I recommend Notepad or TextEdit when you're starting out, but one you have developed your skills a bit, I would recommend switching to Visual Studio Code, which is my favourite one. However, if you prefer another text editor, that's completely fine.













HTML Headings

Headings in HTML are defined using numbers 1-6, with 1 being the largest leading and 2 being the smallest heading. Below is an example of what the headings all look like.

Heading 1

Heading 2

Heading 3

Heading 4

Heading 5
Heading 6

These headings are defined using <h> tags. For example, if you wanted to create a heading that says "Example" in heading size 4, you would write the following code:
<h4>Example<h4>
Writing the code above would result in the output below:

Example


Well done! Feel free to practice headings as much as you want.













HTML Hyperlinks

You can hyperlink text in HTML, similar to Word or Google Docs. This is done by adding the anchor tag (<a>) to the text you would like to hyperlink. (Example below).

<p>This is an example paragraph. I would like to link <a href="https://www.freecodecamp.org/nd03">this</a> to my FCC profile.</p>
This code would result in the following output:

This is an example paragraph. I would like to link this to my FCC profile.

<p>

<li>This is the third item in my ordered list.</li>

Classes and IDs

In HTML, you might want certain <p> elements to look different from other elements. You can do this by using classes and IDs. But what are classes and IDs? A class in HTML is used to change the way that any element looks. You can change the looks of <p>, <header>, <ul> and any other element of your choice. Applying a class to multiple different elements will make all of them look the way that you styled them.
An HTML ID is slightly different. An ID is used to make an element look different from how the element is supposed to look. For example, you can make the size, colour and whatever else you want of a <header> element, and then apply that ID to it. Doing this would make this specific <header> look different from the other <header> elements.
